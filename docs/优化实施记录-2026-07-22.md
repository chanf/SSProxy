# 优化实施记录（2026-07-22）

本记录对应 docs/full-audit-2026-07-21.md、docs/chain-proxy-optimization-review.md 及相关设计文档中的已接受建议。源码仍遵循项目约定：所有交付文件只修改 `build_ipk.py` 的 `src_files`，`src/` 仅由构建生成。

## 已落地

### 服务与网络安全边界

- 自动更新函数使用 `return`，不会退出 `auto_update_loop`。
- Mihomo controller 就绪后才安装 TProxy 和 DNS 劫持规则；核心启动失败时不接管 LAN 流量。
- nftables、UCI、dnsmasq 操作均检查失败状态。
- DNS 劫持保存原始 `noresolv` 和实际端口，启停幂等并在停止时恢复用户配置。
- 白名单按 IPv4/IPv6 分族；缺失某一协议族时显式旁路该族，避免落入 TProxy catch-all。
- ACL IP/CIDR 在进入 nft 规则前由后端校验。

### 配置与数据边界

- `rules: []` 与规则注入流程兼容。
- 数据链路规则使用最终 `run_config` 校验前置节点，避免生成悬空 `ssproxy-chain-*` 规则。
- 端口统一校验为 `1..65535`。
- 订阅下载的 `--resolve` 使用 URL 实际协议和端口，下载过程使用临时文件并保留旧配置备份。
- YAML 双引号字段和 controller secret 统一处理反斜杠、引号及换行。
- JSON 输出统一经过 `json_quote`；覆盖节点、连接、规则、广告源、流量、链路指标、批量延时等接口。
- 批量节点延时测试可读取 JSON 转义后的节点名，不再用不支持引号节点名的简单 grep。

### 输入验证与可观测性

- 访问规则校验域名、动作和规则类型；规则备注写入 UCI。
- 广告源校验名称、URL 协议、behavior 和 format，并在运行配置生成时再次过滤手工修改的非法 UCI 值。
- VMess/VLESS 落地节点校验 `network`。
- 新增 `check_controller`、`wait_controller`、`validate_acl_ip` helper 子命令，并同步 Usage。

### 供应链与下载边界

- Mihomo v1.19.28 的官方 Linux 资产固化 GitHub Release SHA256 digest；自定义核心地址必须使用 HTTPS 并显式提供 64 位 SHA256。
- 核心下载使用系统 CA 校验证书，下载后先校验再解压，使用独立临时目录和原子替换，校验失败不覆盖旧核心。
- GeoIP/GeoSite 与订阅下载移除 `curl -k`；controller Secret 通过 0600 临时 header 文件传给 curl，不出现在进程参数。
- tar/gzip 构建句柄统一由上下文管理器关闭，保持可复现归档格式。

### 采集与链式代理

- `collect_loop` 每 5 秒获取一次 `/connections`，访问日志每 15 秒复用快照，流量与数据链路指标不再重复请求 controller。
- 连接字段改为批量 `jsonfilter` 导出和 awk 聚合，见连接去重与链路指标测试。
- 数据链路健康判定对 `policy` / `chains[0]` 做精确行匹配，不再使用策略名子串搜索。
- `get_chain_status` 返回每条链路的 `runtime`、`intercepted`、`code`、`reason`；白名单旁路可一键加入受控 IP，前置节点切换有重启确认。
- 运行配置只注入被有效启用链路引用的落地节点。

### 多订阅能力

- UCI 从单一 `subscription_url` 扩展为多个匿名 `subscription` 段，设置页支持增删、启停、排序和名称唯一性校验。
- 旧版 `mihomo.config.subscription_url` 自动迁移为第一条订阅，并保留 `/etc/mihomo/.subscription_url` 作为升级兼容。
- `update_subscriptions` 批量更新所有启用订阅，每份订阅独立缓存到 `/etc/mihomo/subscriptions/<sid>.yaml`；单源失败使用旧缓存，所有源都不可用才失败。
- 合并配置以第一份启用订阅为基准，保留其策略组和规则；后续订阅只贡献去重后的节点，同名节点靠前订阅优先。
- 多订阅生成 `SSProxy - 全部订阅` 聚合 select 组，并注入到基准订阅已有 select 组；已兼容块式列表、行内 `proxies: [...]` 和 flow-map 策略组格式。
- 仪表盘显示启用订阅数量，「全部订阅节点」与批量测速覆盖合并后的节点池。

## 残余限制与可选迭代

以下不是当前发布阻塞项：

1. 测试路由器当前无法直连 GitHub，官方核心下载的成功路径需在可访问 GitHub 的设备或镜像环境补验。
2. WS/gRPC/Reality 等传输扩展需要独立的结构化 UCI/UI 设计，不通过自由 YAML 绕过校验。
3. controller 局部热更新、累计流量按日清零、订阅健康报表、helper 拆分属于后续体验和工程演进。

## 发布前建议

每次构建后的 `PKG_VERSION` 自动递增和 `src/` 重建属于项目既定行为。发布前只需在可访问官方 Release 的网络环境补做一次核心下载成功路径，并继续保留失败回滚检查。
